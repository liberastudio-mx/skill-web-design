# Contributing

Contributions are welcome. This is a knowledge base, not a code project — contributions are primarily additions or corrections to design knowledge documents.

## What to contribute

- **Corrections** — factual errors in design principles, outdated browser support info, wrong CSS syntax
- **Additions** — new reference documents covering design domains not yet represented
- **Updates** — keeping browser support percentages, API availability, and tool recommendations current

## What not to contribute

- Aesthetic opinions presented as universal rules
- Framework-specific code that doesn't generalize
- Content that duplicates what's already in the source tools (impeccable, etc.) without adding value

## How to contribute

1. Fork the repository
2. Create a branch: `git checkout -b fix/typography-clamp-example` or `add/dark-mode-patterns`
3. Make your changes
4. Open a pull request with a clear description of what changed and why

## Attribution

If you're contributing content derived from another source, include attribution in your PR description and in the file itself. See [NOTICE.md](NOTICE.md) for how existing attributions are formatted.
