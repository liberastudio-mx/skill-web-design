# Contributing

Contributions are welcome.

This is a knowledge base, not a code project. Contributions are primarily additions, corrections, and updates to design knowledge documents.

## What to contribute

- **Corrections** — factual errors in design principles, outdated browser support information, incorrect CSS syntax, or broken references.
- **Additions** — new reference documents covering design domains not yet represented.
- **Updates** — keeping browser support data, API availability, and tool recommendations current.

## What not to contribute

- Aesthetic opinions presented as universal rules.
- Framework-specific code that does not generalize.
- Content that duplicates existing source material without adding meaningful value.
- Unsourced claims about browser support, APIs, accessibility, or design best practices.
